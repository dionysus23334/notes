操作系统基本概念

操作系统是最基本的系统软件：它管理硬件和软件资源，组织程序运行，并为用户和其他软件提供方便的接口与环境。

1. 定义

操作系统（Operating System，OS）控制和管理整个计算机系统的硬件与软件资源，合理组织计算机工作，调度资源分配，并向用户和其他软件提供方便的接口与运行环境。它是计算机系统中最基本的系统软件。

一句话：OS 站在硬件和应用之间，向下管理资源，向上提供服务。

2. 四大特征

01

并发

多个事件在同一时间间隔内发生。宏观上像是同时进行，微观上可能是交替执行。

02

共享

系统资源可供内存中多个并发执行的进程共同使用。

03

虚拟

把一个物理实体变成若干个逻辑对应物，让用户感受到更丰富的资源。

04

异步

多道程序走走停停，以不可预知的速度向前推进。

3. 并发与并行

并发是两个或多个事件在同一时间间隔内发生。OS 的并发性通常通过分时实现：单核 CPU 同一时刻只能执行一个程序，各程序只能交替执行；多核 CPU 同一时刻可以执行多个程序，因此多个程序可以并行执行。

  • 并发性：两个或多个事件在同一时间间隔内发生。
  • 并行性：两个或多个事件在同一时刻发生，需要多流水线或多处理机等硬件支持。
  • 多道程序环境：宏观上多道程序同时执行；单处理机下，微观上仍是分时交替执行。

4. 共享

共享即资源共享，指系统中的资源可供多个并发执行的进程共同使用。

  • 互斥共享：某些资源可被多个进程使用，但同一时间段只允许一个进程访问。例如 QQ 和微信视频不能同时占用同一个摄像头。
  • 同时共享：某些资源允许一个时间段内由多个进程“同时”访问。例如 QQ 发文件 A、微信发文件 B，宏观上都在读硬盘，微观上可能交替访问。

并发和共享是操作系统最基本的两个特征。资源共享以程序并发为条件；如果系统不能有效管理共享，也会反过来破坏并发执行。

5. 虚拟与异步

虚拟指把一个物理实体变成多个逻辑对应物。物理实体真实存在，逻辑对应物是用户感受到的资源。

  • 虚拟处理器：通过时分复用，让多道程序分时使用一个处理器，使每个用户感觉有 CPU 专门为自己服务。
  • 虚拟存储器：通过空分复用，把物理存储器扩展成逻辑上的更大存储空间。
  • 虚拟 I/O 设备：把一个物理 I/O 设备虚拟成多个逻辑设备，让临界资源变成可共享资源。
  • 异步性：多道程序因资源有限而走走停停，进程推进速度不可预知。

6. 操作系统的功能

为了给多道程序提供良好的运行环境，操作系统要管理处理机、存储器、文件和设备，并提供用户接口。也可以说，操作系统是系统资源的管理者。

CPU

处理机管理

对进程进行管理，包括进程控制、同步、通信、死锁处理和处理机调度。

MEM

存储器管理

包括内存分配与回收、地址映射、内存保护与共享、内存扩充。

FILE

文件管理

通过文件系统管理文件存储空间、目录、文件读写和保护。

I/O

设备管理

完成 I/O 请求，提高设备利用率，包括缓冲管理、设备分配、设备处理和虚拟设备。

7. 向上提供接口

  • 联机命令接口：交互式接口，适用于分时或实时系统。用户说一句，系统做一件并反馈。
  • 脱机命令接口：批处理接口，适用于批处理系统。用户把命令写成清单，系统逐条执行。
  • 程序接口：由系统调用组成，也称广义指令，是给程序员使用的接口。普通用户通常通过程序代码间接使用。
  • GUI：图形化用户接口，让用户用图形界面操作，不必记忆复杂命令和参数。

8. 最接近硬件的软件

没有任何软件支持的计算机称为裸机，它只构成计算机系统的物质基础。在裸机上安装操作系统后，系统获得资源管理和方便用户使用的服务能力。

覆盖了软件的机器通常称为扩充机器，也称虚拟机。OS 的价值，就是把难用的裸机改造成更强、更方便、更高效的机器。
